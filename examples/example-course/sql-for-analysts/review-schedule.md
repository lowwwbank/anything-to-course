# Review Schedule

Reviews are **retrieval sessions** — answer from memory first, check second. Rereading lessons does not count as a review. Each session is 10–15 minutes.

| When | What | How |
|---|---|---|
| Right after each lesson | that lesson's exit ticket | already built into the lessons |
| **+1 day** after module 1 | write from memory: the 3-clause query anatomy; the NULL rule; one query with mixed AND/OR | blank page, then check against [answers/module-1.md](answers/module-1.md) |
| **+1 day** after module 2 | write from memory: the 6-step processing order; one GROUP BY + HAVING query for a question you invent | then check section 2.1 feedback |
| **+7 days** | redo module 1 checkpoint tasks 3–5 *from the task text only* | compare with your first attempt — note what improved and what didn't |
| **+7 days** (module 2) | redo cumulative checkpoint tasks 2–3 with these swaps: Japan→Brazil, coupons→statuses | swapped surfaces test the principle, not the memory of the answer |
| **+21–30 days** | the transfer task (checkpoint 2, task 5) on a *third* table you sketch yourself from your real work | if you can generate and solve your own variant, the capability is yours |
| **+2–3 months** (optional, if SQL matters long-term) | one self-invented per-group question against a real table at work | real-world closure |

## Calibration log

After every session, note tasks where confidence said **sure** but the answer was wrong. Two misses on the same topic → that topic jumps to tomorrow regardless of schedule.

## When you finish the course

Finishing ≠ done: the +7 and +21–30 day sessions are where the knowledge consolidates. Put them in your calendar now — future-you won't do it spontaneously.
